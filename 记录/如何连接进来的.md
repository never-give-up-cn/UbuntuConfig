## 操作步骤

### 1. 确认目标主机可达

```bash
# 使用 ping 检查网络连通性
ping -n 2 -w 3000 192.168.183.131
```

> 结果：目标可达，延迟 <1ms

### 2. 尝试命令行 SSH 连接

由于当前环境为 Windows (无图形界面)，需要以非交互方式传递密码。尝试了以下方法：

#### 方法一：plink（PuTTY 命令行工具）

```bash
plink -ssh -batch -pw '1' pi@192.168.183.131 "command"
```

- **问题**：主机密钥未缓存，`-batch` 模式下无法交互接受密钥
- **解决**：添加 `-hostkey` 参数指定主机密钥指纹

```bash
plink -ssh -batch -pw '1' -hostkey "SHA256:O5ySIS29umaZFQoELneOYpBOkIy46obL6gjAqZH4Yxs" pi@192.168.183.131 "command"
```

#### 方法二：OpenSSH + SSH_ASKPASS

设置 `SSH_ASKPASS` 环境变量指向密码输出脚本：

```bash
export SSH_ASKPASS=/path/to/password_script.sh
export DISPLAY=none
ssh -T -o StrictHostKeyChecking=no pi@192.168.183.131 "command"
```

- **问题**：Windows 环境中 `SSH_ASKPASS` 未生效，SSH 仍尝试读取 `/dev/tty`

#### 方法三：Python + paramiko 库（最终成功方案）

```python
import paramiko

client = paramiko.SSHClient()
client.set_missing_host_key_policy(paramiko.AutoAddPolicy())
client.connect("192.168.183.131", 22, "pi", "1", timeout=20)
stdin, stdout, stderr = client.exec_command("command")
print(stdout.read().decode())
client.close()
```

### 3. 遇到的障碍

#### 障碍 1：MaxStartups 连接频率限制

- **现象**：多次连接后返回 `"Not allowed at this time"` 或连接超时
- **原因**：OpenSSH 的 `MaxStartups` 机制，短时间内大量未认证连接会被丢弃
- **解决**：每次连接间等待 3-5 秒，避免频繁连接；若被限制则等待 30-60 秒恢复

#### 障碍 2：非交互式密码传递

- **现象**：SSH 客户端需要 TTY 读取密码，但在非交互环境中无法获取
- **解决**：使用 Python paramiko 库，以编程方式传递凭据

#### 障碍 3：Windows 编码问题

- **现象**：Unicode 字符（如 emoji）在 GBK 编码下无法打印
- **解决**：使用 `errors='replace'` 处理编码异常

## 最终连接命令（快速参考）

```python
import paramiko

client = paramiko.SSHClient()
client.set_missing_host_key_policy(paramiko.AutoAddPolicy())
client.connect("192.168.183.131", 22, "pi", "1", timeout=20, look_for_keys=False, allow_agent=False)

# 执行命令
stdin, stdout, stderr = client.exec_command("hostname")
print(stdout.read().decode())

client.close()
```

## 重要说明

- **MaxStartups 限制**：每次连接失败后建议等待至少 3 秒再重试
- **密码为纯数字 "1"**，注意不要与特殊字符混淆
- **主机密钥指纹**：`SHA256:O5ySIS29umaZFQoELneOYpBOkIy46obL6gjAqZH4Yxs`（ED25519）
